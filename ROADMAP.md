# SmartTennis 路线图 / Roadmap

> 把"上传一段视频→出诊断"的 MVP，做成一个真正帮球友变强、且有壁垒的产品。

## 一、现在在哪（MVP 现状）

- **能力**：单拍种（正手）、单视角手机视频 → MediaPipe 2D 骨骼 → 动力链诊断 + 影子对比 + **击球帧校正滑杆**。
- **已修复的信任问题**：contact 检测（前挥向前速度峰，替代会选到引拍的旧逻辑）、X-factor 垃圾值（wrap 到 ±180°）、**连续评分**（不再动辄 100）、**发力顺序数字透明化**（去黑箱）。
- **已知边界（要诚实说）**：
  - 单视角深度噪声 → X-factor 等"绕垂直轴"的量仍有误差（已有界，非毫米级）。
  - 自动 contact 有 ~±0.4s 固有误差（单手机单视角的物理极限）→ 用滑杆兜底。
  - 参考库目前**单一来源**（德约一段慢动作切的 87 条），代表性有限。

## 二、护城河在哪（反复确认）

**代码不是壁垒**（MediaPipe 开源、生物力学公开，任何人都能复刻一个 demo）。真正的壁垒：

1. **职业动作参考库**：多球员 × 多机位 × 多拍种、且逐条验证过的高质量数据集。
2. **"测得准 + 真能帮人提高"的诊断引擎与信任**。
3. **用户数据飞轮**：越多人用 → 数据越多 → 诊断越准。
4. **教练内容 / 品牌 / 分发渠道**。

## 三、分阶段计划

### Phase 1 — 参考库质量（最高优先，进行中）
- [ ] **多机位/多届德约**：削弱单视角、单 session 偏差（正在下载处理 court-level 2014/2021/2023、US Open 2023）。
- [ ] **多球员 → 风格档（archetype）**：平击进攻型 / 重上旋型 / 全面型，球员不够多时先按风格聚类。
- [ ] **重构参考结构**：拆成
  - 「**通用基本功库**」= 多球星 pool（发力时序、近端→远端顺序等跨人通用量）——驱动"打对了没"的诊断；
  - 「**单人/风格对照库**」= 按球员/风格分——驱动"你像谁/你想学谁"的视觉对照与营销。
- [ ] **数据清洗**：自动剔除切歪、检测质量差、非正手的 stroke（提高 band 质量）。
- [ ] **指标尽量用归一化/相对量**（时序、顺序），少用绝对速度/力量（业余学不来、伤身）。
- [ ] **合成"范本标准动作"骨架（R&D，多球星数据到位后做）**：把多条好挥拍 DTW/相位对齐 → 在归一化空间里**平均关节角度**（不是坐标，避免身材/风格平均成扭曲怪人）→ 重建一个干净的"标准动作"骨架。三个红利：① **版权干净**（是算出来的数据，不是任何人的视频）；② 代表多球星**共识**，比单条视频更适合当纠错样板；③ 是 3D 角度 → **可投影任意视角**（接"任意角度匹配用户"）。诊断仍针对"共性基本功"的偏离，不强求复刻单一姿势。

### Phase 2 — 更多拍种
- [ ] 反手、发球、截击；复用同一套动力链框架；每个拍种一套参考库。
- [ ] 自动拍种识别（先简单分类器或规则，后续可上小模型）。

### Phase 3 — 多模型融合（球拍 / 球）
- [x] **击球点（前伸/高度）已先用姿态+3D实现**：在解剖坐标系量持拍手腕相对髋的前伸量与高度（视角无关、身材归一化），**无需重型 ML**。已上线为 2 个诊断项（"击球点·前伸/高度"）。
- [ ] **球拍**：目标检测（YOLO，COCO 自带 `tennis racket` 类）→ 球拍位置 → 精化击球点 + 拍面朝向。
- [ ] **球**：TrackNet（专为小、快、糊的球设计）→ 球轨迹 → **真·contact**（球方向突然反转那一帧）、进而球速/旋转估计。
- [ ] **拍摄引导**：提示用户用手机**慢动作（120/240fps）**拍——几乎零成本却能大幅提升球拍/球检测与 contact 精度。
- ⚠️ **重型 ML 是一次明确的基建决定**：引入 torch/YOLO/TrackNet 会让镜像暴涨(~GB级)、小 VPS 部署变难；单手机低清追球本就不可靠。**故先用姿态把能做的价值做满，重型 ML 待你确认后再上**（可考虑只在更强的服务器/GPU 或离线批处理里做）。

### Phase 4 — 产品化（启动数据飞轮）
- [ ] 用户账号、历史记录、**进步追踪**（同一指标随时间的曲线）。
- [ ] 训练计划 / drill 库，与每条诊断结论挂钩（"这个毛病 → 练这个动作"）。
- [ ] 分享卡片（你 vs 偶像的影子对比），自带传播。

### Phase 5 — 分发
- [ ] 教练 / 网校 / 俱乐部合作；B 端：教练用它给学员出可视化报告。
- [ ] 内容获客：偶像对照 + 短视频。

## 四、怎么证明"真有用"（投资人一定会问）

- 找 **10–50 个真实球友**持续用，用"前后对比 + 教练主观评分"验证诊断是否真带来改进。
- 轻量 A/B：按我们的建议练 vs 不练，看目标指标是否改善。
- **这个证据比任何技术名词都打动投资人**——早期看的是团队、市场、真实需求与一个"小而尖"的切入点。

## 五、技术债 / 待办

- [ ] 单视角 → 评估双机位或单视角 3D-lift，提升 X-factor 可靠性。
- [ ] contact 精确化（引入球拍/球）。
- [ ] 评分 band 用更多数据校准，避免过宽（不区分）或过窄（误杀）。
- [ ] 隐私：用户上传视频的存储/删除策略（当前是临时文件，会话淘汰即删）。
- [ ] 部署：国内 VPS 上线（已就绪），后续考虑域名+备案、HTTPS（启用实时取流 getUserMedia 需要）。
